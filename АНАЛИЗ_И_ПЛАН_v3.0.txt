✅ ЧТО РАБОТАЕТ ХОРОШО (текущий промпт):

1. Железные правила не нарушаются
   ✓ Контакты перед тарифами
   ✓ Только данные из базы знаний
   ✓ Никаких личных скидок
   ✓ Честная аргументация (не обещает заработки)

2. Структура воронки четкая
   ✓ 10 шагов от приветствия до оплаты
   ✓ Антисогласие (если клиент называет неверную цену)
   ✓ Отработана работа с возражениями

3. Защита от вранья хорошая
   ✓ "Никогда не считай рассрочку сама"
   ✓ "Если данных нет — уточню"
   ✓ "Проверь и поправь" при неверных цифрах

❌ ЧТО НУЖНО УЛУЧШИТЬ:

1. СПАМ НАПОМИНАНИЙ (критичная проблема)
   
   СЕЙЧАС:
   - 5 сообщений за 21 час
   - Отправляются даже во время активного диалога
   - Продолжаются даже после менеджера (может быть)
   - Одинаковые тексты повторяются
   - Выглядит как мусор в диалоге
   
   ДИАГИ ИЗ РЕАЛЬНЫХ ДИАЛОГОВ:
   [00:14] Клиент: Я подумаю
   [00:14] Луна: Хорошо!
   [03:14] Луна: Напоминание #1 ← нормально
   [08:14] Луна: Напоминание #2 ← нормально
   [12:46] Менеджер: Добрый день
   [17:47] Луна: Напоминание #3 ← СПАМ! Менеджер уже взял чат
   
   Еще хуже:
   [01:09] Клиент: нужно время, хотелось бы попозже
   [01:09] Луна: Можем зафиксировать...
   [02:09] Луна: НАПОМИНАНИЕ ← во время АКТИВНОГО диалога!

2. МЕХАНИЧЕСКИЙ ТОН
   
   Текущий: "Цены периодически меняются, поэтому лучше не откладывать надолго"
   Звучит как: давление, робот, продавец
   
   Нужно: "Цены периодически меняются, а вы не потеряете цену если забронируете"
   Звучит как: помощь, понимание, друг

3. ОТСУТСТВИЕ АДАПТИВНОСТИ
   
   Текущий промпт не инструирует как:
   • Адаптировать тон под эмоции клиента
   • Распознать страх vs просто занятость vs незаинтересованность
   • Менять подход в зависимости от реакции
   
   DeepSeek это добавил, но мы должны без рисков на обещания заработков

4. ВИДЕО И КЕЙСЫ
   
   Сейчас: генерируются напоминания с видео
   Проблема: видео выбираются случайно, могут быть не релевантны
   Нужно: чётко определить какое видео при каком шаге

📋 ШАГ 2: ИЗМЕНЕНИЯ В BOT.PY (код напоминаний)

ТЕКУЩАЯ ЦЕПОЧКА (ПЛОХАЯ):
┌─────────────────────────────────────────────┐
│ REMINDER_CHAIN = [                          │
│   (3600,    "Иногда одно решение..."),     │ ← 1 час
│   (10800,   "Вчера заметили..."),          │ ← +3 часа (итого 4)
│   (18000,   "Иногда достаточно..."),       │ ← +5 часов (итого 9)
│   (36000,   "Многие ученики..."),          │ ← +10 часов (итого 19)
│   (75600,   "Можно вопрос..."),            │ ← +21 час (итого 40)
│ ]                                           │
└─────────────────────────────────────────────┘

НОВАЯ ЦЕПОЧКА (ХОРОШАЯ):
┌─────────────────────────────────────────────────────────────┐
│ REMINDER_CHAIN = [                                          │
│   (3600,    "Если сейчас неудобно..."),    │ ← Вариант А │ ← 1 час
│   (21600,   "Знаете, большинство..."),    │ ← Вариант В │ ← +5 часов
│   (86400,   "Может быть, помогут..."),    │ ← Вариант С │ ← +24 часа
│ ]                                           │                  │
└─────────────────────────────────────────────────────────────┘

ИТОГО: 3 вместо 5, последнее за 26 часов вместо 40+

ИСПРАВЛЕНИЯ В КОД:

1. Заменить REMINDER_CHAIN в bot.py на новую версию (выше)

2. Добавить проверку: НЕ отправлять напоминание если:
   └─ Клиент ответил на ПОСЛЕДНЕЕ сообщение Луны менее часа назад
   └─ STATE_MANAGER (менеджер взял чат)
   └─ Клиент ответил на напоминание (даже "подумаю")

3. Убедиться что cancel_reminder_chain() вызывается при:
   └─ set_state(..., STATE_MANAGER)
   └─ Менеджер пишет первый ответ

ПРОВЕРКА КОД (строки в текущем bot.py):
  1397: cancel_reminder_chain() при ответе менеджера ← ✅ ДОЛЖНО БЫТЬ
  1810: start_reminder_chain(REMINDER_CHAIN_MANAGER_START_STEP) ← ❌ УБРАТЬ ЭТУ!
  1843: start_reminder_chain(REMINDER_CHAIN_MANAGER_START_STEP) ← ❌ УБРАТЬ ЭТУ!

  Объяснение: если менеджер ответил → cancel_reminder_chain(),
  а потом снова start_reminder_chain(MANAGER_STEP) — это дублирует напоминания!
