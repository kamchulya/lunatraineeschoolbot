# Что изменилось в этом заходе

## bot.py
- Исправлен маркер `detect_unknown_answer()` — раньше искал фразу, которой
  нет в промпте, поэтому уведомление Артёму по неизвестным вопросам почти
  никогда не срабатывало. Теперь ловит реальные фразы из prompt.py.
- Исправлен баг в `claude_reply()` — подряд идущие сообщения одной роли
  (например несколько сообщений клиента, пока чат в STATE_MANAGER) раньше
  тихо ВЫБРАСЫВАЛИСЬ из контекста модели. Теперь склеиваются, а не теряются.
- Добавлен Сценарий 1 для комментариев в Instagram: обычные комментарии
  (не "хочу") получают ответ через Haiku (вопрос → отвечает по существу,
  похвала без вопроса → благодарит), комментарии из одних эмодзи — игнор.
- Добавлен перенос сделки на этап «Чат-бот» в CRM (`move_deal_to_chatbot_stage`,
  `update_deal_stage`) — метод `/openapi/v1/deal/updateDealStage` подтверждён
  рабочим кодом бота Лолы. ID этапов зашиты для двух воронок (ВХОДЯЩИЕ и
  Школа Фитнеса).
- Добавлена отправка договора: страница `/dogovor`, статика `/static/dogovor.pdf`,
  функция `send_wazzup_file()` для отправки самого PDF в чат (⚠️ не проверена
  вживую — см. пометку в коде), хелпер `get_knowledge_base()` подмешивает
  ссылку на договор в базу знаний, если задан `PUBLIC_BASE_URL`.

## prompt.py
- Уточнение про экзамен на свидетельство о профквалификации — процент сдачи
  определяет лектор, единой цифры нет.
- Правило по гео/2ГИС и видео-обзору зала (видео — только по запросу
  клиента, не проактивно).
- Шаг 10 (Оплата): договор отправляется ПЕРЕД ссылкой на оплату.

## knowledge_base.py
- Добавлены 2ГИС-ссылки и видео-обзоры зала для всех 4 адресов
  (Туран 50, Мухамедханова 21, Бухар Жырау 25/1, Aquastars Алматы).
- Уточнение про экзамен (см. выше).

## config.py
- Новая переменная `PUBLIC_BASE_URL` — нужно задать в Railway Variables
  (домен сервиса без https:// и без слэша на конце), иначе бот не будет
  упоминать договор.

## static/ (новая папка)
- `dogovor.html` — веб-страница с текстом договора.
- `dogovor.pdf` — оригинал договора (тот файл, что ты присылала).

## Что нужно сделать в Railway после деплоя
1. Settings → Variables → добавить:
   ```
   PUBLIC_BASE_URL=web-production-5bd41.up.railway.app
   ```
2. Проверить `https://web-production-5bd41.up.railway.app/dogovor` открывается.
3. Проверить в логах после первого реального ответа бота (не напоминания):
   строку `🏷️ deal/updateDealStage ... [200]` — и глазами в EnvyCRM, что
   карточка правда переехала на этап «Чат-бот».
4. Если попросишь бота прислать PDF в чат — проверить строку `📎 Wazzup file →`
   и глазами в Instagram, что файл реально дошёл (это непроверенная часть).
