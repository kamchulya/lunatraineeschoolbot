## 🎯 Основная проблема

В логах обнаружена ошибка:
```
ERROR ❌ Claude error: 'ThinkingBlock' object has no attribute 'text'
```

**Причина:** DeepSeek иногда возвращает `ThinkingBlock` (блок с внутренним мышлением), но старый код попытался получить `.text` сразу из `msg.content[0]` без проверки типа блока → крах → фолбэк "Извините, небольшой сбой".

**На клиенте проявляется:** вместо нормального ответа бот отправляет ошибку, клиент видит что бот "сбоит".

---

## ✅ Что исправлено

### 1. **bot.py** — Полная переработка `claude_reply()` функции

#### Новая логика fallback (4 попытки):

```
DeepSeek попытка 1 (15сек)
  ├─ ✅ успех → вернуть ответ
  └─ ❌ пусто/timeout/exception → переход на попытку 2

DeepSeek попытка 2 (15сек)
  ├─ ✅ успех → вернуть ответ
  └─ ❌ пусто/timeout/exception → переключение на OpenAI

OpenAI попытка 1 (15сек) — только если DeepSeek дважды пуст
  ├─ ✅ успех → вернуть ответ
  └─ ❌ пусто/timeout/exception → переход на попытку 2

OpenAI попытка 2 (15сек)
  ├─ ✅ успех → вернуть ответ
  └─ ❌ пусто/timeout/exception → фолбэк-текст
```

#### Ключевые изменения:

**1. Отключен thinking в DeepSeek:**
```python
extra_body={"thinking": {"type": "disabled"}}
```
Это убирает `ThinkingBlock` из ответа → всегда получается `TextBlock`.

**2. Правильная обработка content блоков:**
```python
text_parts = []
for block in msg.content:
    if hasattr(block, 'text'):  # Только текстовые блоки
        text_parts.append(block.text)
result = "".join(text_parts).strip()
```

**3. Timeout 15 сек на оба провайдера:**
```python
await asyncio.wait_for(client.messages.create(...), timeout=15.0)
```
Если долго думает → timeout → переключение на OpenAI.

**4. Timeout и exception обрабатываются одинаково:**
Нет отдельной ветки для ошибок — всё как пустой content = переход на следующую попытку.

**5. Два вспомогательных метода:**
- `_call_deepseek_once()` — одна попытка DeepSeek
- `_call_openai_once()` — одна попытка OpenAI (gpt-5-mini)

**6. Логирование для отладки:**
- `log.info` при успехе
- `log.warning` при переключениях и повторах
- `log.error` при финальном фолбэке

**7. Интеллектуальное определение языка:**
В фолбэк-текст используется язык последнего сообщения клиента (ищет казахские буквы әіңғүұқө).

## 🚀 Как это работает в реальности

### Сценарий 1: DeepSeek быстро отвечает
```
Клиент → "Хочу на курс" 
  → Claude звонит DeepSeek
  → DeepSeek отвечает за 2сек с thinking=disabled
  → TextBlock → .text возвращается
  → Клиент получает ответ за 3сек ✅
```

### Сценарий 2: DeepSeek слишком долго думает
```
Клиент → "Какие тарифы?" 
  → Claude звонит DeepSeek (timeout 15сек)
  → DeepSeek молчит 16сек → timeout
  → Повтор DeepSeek (timeout 15сек)
  → Снова пусто → переключение на OpenAI
  → OpenAI отвечает за 3сек (gpt-5-mini быстро)
  → Клиент получает ответ за 35сек (15+15+3+2 макс) ✅
```

### Сценарий 3: Обе системы не работают (редко)
```
Клиент → "Что-то"
  → DeepSeek x2 пусто
  → OpenAI x2 пусто
  → Фолбэк на казахском/русском ✅
  → Клиент: "Извините, қате болды" или "Извините, небольшой сбой"
```

## ⚠️ Важные примечания

1. **thinking=disabled** — это параметр который только DeepSeek понимает. Для Anthropic Claude это не нужно.
2. **timeout 15сек** — достаточен для обоих провайдеров, DeepSeek v4-flash обычно отвечает за 1-5сек, OpenAI gpt-5-mini за 1-3сек.
3. **OpenAI только при нужде** — OpenAI не вызывается если DeepSeek успешен, экономим токены.
4. **max_completion_tokens для OpenAI** — это правильный параметр для gpt-5-mini, НЕ max_tokens.
5. **system_prompt для OpenAI как строка** — не список блоков как в DeepSeek, поэтому склеиваем в одну строку.
